## 1. 测试用例设计

### 1.1 功能测试用例

| 编号 | 测试场景 | 预期结果 | 优先级 |
|------|----------|----------|--------|
| TC-01 | 用户输入一段 B 站视频URL/文案素材 | 系统成功接收素材并展示预览 | P0 |
| TC-02 | 点击"生成文案"按钮 | 系统调用 AI 生成文案，返回1-3个选项 | P0 |
| TC-03 | 点击"生成标题"按钮 | 系统调用 AI 生成标题，返回3-5个选项 | P0 |
| TC-04 | 点击单个文案卡片 | 文案被选中，高亮显示 | P1 |
| TC-05 | 点击单个标题卡片 | 标题被选中，高亮显示 | P1 |
| TC-06 | 点击"一键复制"按钮 | 选中的文案+标题复制到剪贴板，提示"已复制" | P0 |
| TC-07 | 未选择文案时点击复制 | 提示"请先选择文案" | P1 |
| TC-08 | 网络请求超时 | 显示友好错误提示，不闪退 | P1 |
| TC-09 | AI 生成失败 | 显示错误信息，提供重试按钮 | P1 |

### 1.2 边界测试用例

| 编号 | 测试场景 | 预期结果 | 优先级 |
|------|----------|----------|--------|
| TC-10 | 输入为空或仅空格 | 提示"请输入素材内容" | P0 |
| TC-11 | 输入超过5000字 | 提示"素材内容过长，请控制在5000字以内" | P1 |
| TC-12 | 输入特殊字符/emoji | 正常处理，不崩溃 | P2 |
| TC-13 | 快速连续点击生成按钮 | 防重复提交，显示加载状态 | P1 |
| TC-14 | 复制按钮连续点击 | 防抖处理，避免多次复制 | P2 |

### 1.3 用户体验测试用例

| 编号 | 测试场景 | 预期结果 | 优先级 |
|------|----------|----------|--------|
| TC-15 | 首次加载页面 | 骨架屏/加载动画不超时 | P1 |
| TC-16 | 生成过程中 | 显示加载动画和进度提示 | P1 |
| TC-17 | 生成完成后 | 结果区域平滑展开 | P2 |
| TC-18 | 复制成功 | Toast 提示持续2秒后消失 | P2 |

## 2. 验收标准清单

### 2.1 功能验收

- [ ] **素材输入**：支持文本输入，URL 自动识别
- [ ] **文案生成**：AI 生成 1-3 条文案，响应时间 < 5秒
- [ ] **标题生成**：AI 生成 3-5 个标题，响应时间 < 3秒
- [ ] **一键复制**：文案+标题组合复制，成功率 100%
- [ ] **错误处理**：各类异常有友好提示，不闪退

### 2.2 性能验收

| 指标 | 目标值 | 测试方法 |
|------|--------|----------|
| 首屏加载 | < 2秒 | Lighthouse / Web Vitals |
| API 响应 (生成文案) | < 5秒 | 计时工具 |
| API 响应 (生成标题) | < 3秒 | 计时工具 |
| 复制功能 | < 500ms | 计时工具 |

### 2.3 兼容性验收

- [ ] iOS Safari 最新版
- [ ] Android Chrome 最新版
- [ ] 微信内置浏览器
- [ ] 桌面端 Chrome/Firefox/Safari

### 2.4 视觉验收

- [ ] 页面布局正确，无溢出/错位
- [ ] 按钮可点击区域 ≥ 44x44px
- [ ] 文字可读性符合 WCAG AA 标准
- [ ] 加载状态明确可见
